# 技术创新点说明

本项目在基础 SSM 框架之上，实现了多项技术创新点，提升了系统的安全性、可维护性和用户体验。

---

## 一、统一响应封装（Result 类）

**文件位置**：`com.campus.common.Result`

**技术说明**：
- 所有 API 接口统一返回 `Result<T>` 对象
- 包含 `success`、`code`、`message`、`data`、`timestamp` 字段
- 便于前端统一处理响应，提高代码规范性

**代码示例**：
```java
public static <T> Result<T> success(T data) {
    Result<T> result = success();
    result.setData(data);
    return result;
}

public static <T> Result<T> error(Integer code, String message) {
    Result<T> result = new Result<>();
    result.setSuccess(false);
    result.setCode(code);
    result.setMessage(message);
    return result;
}
```

## 二、全局异常处理器

**文件位置**：`com.campus.exception.GlobalExceptionHandler`

**技术说明**：
- 使用 `@ControllerAdvice` + `@ExceptionHandler` 实现全局异常统一处理
- 避免在每个 Controller 中重复编写 try-catch
- 自动记录异常日志，便于问题排查

**核心原理**：
```java
@ControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(BusinessException.class)
    @ResponseBody
    public Result<Object> handleBusinessException(BusinessException e) {
        logger.warn("业务异常: {}", e.getMessage());
        return Result.error(e.getCode(), e.getMessage());
    }
    
    @ExceptionHandler(Exception.class)
    @ResponseBody
    public Result<Object> handleException(Exception e) {
        logger.error("系统异常: {}", e.getMessage(), e);
        return Result.error("系统繁忙，请稍后重试");
    }
}
```

## 三、AOP 操作日志（声明式日志记录）

**文件位置**：
- `com.campus.annotation.Log`（自定义注解）
- `com.campus.aspect.LogAspect`（切面类）

**技术说明**：
- 使用 Spring AOP + 自定义注解实现声明式日志记录
- 只需在方法上加 `@Log("操作描述")` 即可自动记录
- 自动获取用户信息、请求参数、执行时长、IP地址等

**使用方式**：
```java
@Log("用户登录")
@RequestMapping(value = "/login", method = RequestMethod.POST)
public Map<String, Object> login(String username, String password) {
    // 方法执行前后会自动记录日志
}
```

**日志输出示例**：
```
【操作日志】用户: admin, 操作: 用户登录, 方法: UserController.login(), 耗时: 35ms, IP: 127.0.0.1
```

**文件位置**：

## 七、Spring 声明式事务

**文件位置**：`com.campus.service.impl.OrderServiceImpl`

**技术说明**：
- 使用 `@Transactional` 注解实现声明式事务管理
- 订单创建和商品状态更新在同一个事务中
- 保证数据一致性，防止超卖

```java
@Transactional(rollbackFor = Exception.class)
public boolean createOrder(Order order) {
    // 1. 检查商品状态（防止超卖）
    Product product = productMapper.findById(order.getProductId());
    if (product.getStatus() != 0) {
        throw new RuntimeException("商品已售出");
    }
    
    // 2. 创建订单
    orderMapper.insert(order);
    
    // 3. 更新商品状态
    productMapper.updateStatus(order.getProductId(), 1);
    
    // 以上操作要么全部成功，要么全部回滚
    return true;
}
```

## 九、文件存储分离

**文件位置**：`com.campus.util.FileUploadUtil`

**技术说明**：
- 图片不存入数据库（二进制），而是存储在服务器文件系统
- 数据库中只存储图片的相对路径
- 使用 UUID 生成唯一文件名，防止文件名冲突

**优点**：
1. 减少数据库存储压力
2. 图片可以通过静态资源方式访问，性能更好
3. 便于后期迁移到 CDN 或对象存储

| 文件存储分离 | 文件流操作 | 优化存储性能 |

这些技术点不仅展示了对 SSM 框架的深入理解，也体现了对企业级应用开发中常见问题的解决能力。
