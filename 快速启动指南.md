# 快速启动指南

## 一、环境准备

1. **安装JDK 1.8+**
   - 下载并安装JDK
   - 配置JAVA_HOME环境变量

2. **安装Maven 3.6+**
   - 下载并安装Maven
   - 配置MAVEN_HOME环境变量

3. **安装MySQL 5.7+ 或 8.0+**
   - 下载并安装MySQL
   - 启动MySQL服务

4. **安装Tomcat 8.5+**
   - 下载并解压Tomcat

> **若使用下文「Docker Compose 一键启动」**：可不在本机单独安装 MySQL / Redis / Tomcat（由容器提供）；仍需安装 **Docker Desktop**（Windows/macOS）或 Docker Engine + Compose 插件（Linux）。

## 二、Docker Compose 一键启动（可选 · 多实例）

适用于**第一阶段多实例验收**：一次启动 **MySQL + Redis + MinIO + 两个 Tomcat（8080 / 8081）**，两实例共用 Redis Session，无需本机先建库（首次会自动执行 `docker/init/01-schema.sql` 初始化）。

### 1. 前置条件

- 已安装并启动 [Docker Desktop](https://www.docker.com/products/docker-desktop/)（或 Linux 下的 Docker + `docker compose`）。
- 本机 **8080、8081、3307、6380、9000、9001** 未被其它程序占用（若 6379 已被本机 Redis 占用不影响，Compose 内 Redis 映射在 **6380**）。

### 2. 启动与停止

在项目根目录执行其一即可：

- **Windows**：双击 `start.bat`，或 PowerShell / CMD：`docker compose up -d --build`
- **Linux / macOS**：`chmod +x start.sh && ./start.sh`，或：`docker compose up -d --build`

停止并删除容器（保留数据卷）：

```bash
docker compose down
```

停止并**清空** Compose 管理的 MySQL 等数据卷（初始化脚本会重新跑，用于修乱码或重来）：

```bash
docker compose down -v
```

### 3. 访问地址与账号

| 说明 | 地址或参数 |
|------|------------|
| 应用实例 1 | http://localhost:8080 |
| 应用实例 2 | http://localhost:8081 |
| MySQL（映射到本机） | 主机 `localhost`，端口 **3307**，库名 `secondhand_market`，用户 `root`，密码 **`campus_root`** |
| Redis（从本机连容器内实例） | **localhost:6380**（映射到容器内 6379；应用容器内仍使用服务名 `redis:6379`） |
| MinIO API | http://localhost:9000 |
| MinIO 控制台 | http://localhost:9001 ，账号密码默认 **`minioadmin` / `minioadmin`** |
| 管理员（库初始化脚本写入） | 用户名 **`admin`**，密码 **`admin123`** |

### 4. 与本地 `mvn` 方式的区别（简要）

- 连接信息在构建时由 **`docker/campus-db.properties`**、**`docker/campus-redis-config.properties`** 打入 WAR，**不要**与 `src/main/resources/db.properties` 混用；本机改 `db.properties` 不会影响已构建的镜像，改 Docker 配置后需 **`docker compose build --no-cache`** 再启动。
- 镜像内为 **Tomcat 9**（避免 Tomcat 7 与 Spring 5.3 静态资源等兼容问题）。
- 上传目录在容器内为 **`/data/upload/`**，两 Tomcat 挂载同一数据卷，便于本地上传回退路径在多实例间共享。

### 5. 多实例 Session 验收建议

1. 浏览器访问 **8080**，注册或登录。  
2. 再打开 **8081** 同一路径，应仍为已登录（Session 存 Redis）。  

### 6. 常见问题（Docker）

- **容器名已存在**：删除旧容器，例如 `docker rm -f campus-minio`（若曾使用固定 `container_name`）；当前 Compose 项目名为 **`campus-secondhand`**，一般不再固定容器名。  
- **端口被占用**：按报错修改 `docker-compose.yml` 中对应 `ports` 左侧宿主机端口。  
- **分类等中文乱码**：确保使用最新 `docker/campus-db.properties`（含 `UTF-8` 与 `utf8mb4` 排序规则）与 `docker/init/01-schema.sql`（含 `SET NAMES utf8mb4`），重新构建后若库中已是错误数据，执行 **`docker compose down -v`** 再 **`docker compose up -d --build`**。

相关文件：`Dockerfile`、`docker-compose.yml`、`docker/init/01-schema.sql`、`docker/campus-*.properties`、`start.bat` / `start.sh`。更详细的说明见 **`README.md`** 中「Docker 一键启动」小节。

## 三、数据库配置

1. **创建数据库**
   ```sql
   CREATE DATABASE secondhand_market DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

2. **执行SQL脚本**
   ```bash
   mysql -u root -p secondhand_market < src/main/resources/db/schema.sql
   ```
   或者直接在MySQL客户端中执行 `src/main/resources/db/schema.sql` 文件

3. **修改数据库连接配置**
   编辑 `src/main/resources/db.properties`：
   ```properties
   jdbc.username=你的MySQL用户名
   jdbc.password=你的MySQL密码
   ```

## 四、文件上传路径配置

编辑 `src/main/resources/db.properties`，设置文件上传路径：

**Windows系统：**
```properties
upload.path=D:/upload/
```

**Linux/Mac系统：**
```properties
upload.path=/tmp/upload/
```

**重要：** 请确保该目录存在且有写权限！

## 五、编译打包

在项目根目录执行：
```bash
mvn clean package
```

打包成功后，会在 `target` 目录生成 `secondhand-market.war` 文件。

> **仅使用 Docker 部署时**：`Dockerfile` 构建阶段会执行 `mvn package`，可不先在宿主机执行本节；若需本地调试 WAR，仍建议执行一次以便在 `target` 目录拿到产物。

## 六、部署运行

### 方式一：使用Tomcat部署

1. 将 `target/secondhand-market.war` 复制到 Tomcat 的 `webapps` 目录
2. 启动Tomcat（运行 `bin/startup.bat` 或 `bin/startup.sh`）
3. 访问 `http://localhost:8080/secondhand-market/`

### 方式二：使用Maven插件运行（开发环境）

在项目根目录执行：
```bash
mvn tomcat7:run
```

然后访问 `http://localhost:8080/`

### 方式三：Docker Compose（见第二节）

一键多实例与中间件，命令与端口见 **「二、Docker Compose 一键启动」**。

## 九、常见问题

### 1. 数据库连接失败
- 检查MySQL服务是否启动
- 检查 `db.properties` 中的用户名密码是否正确
- 检查数据库是否已创建

### 2. 文件上传失败
- 检查 `upload.path` 配置是否正确
- 检查上传目录是否存在且有写权限

### 3. 页面404错误
- 检查Tomcat是否正常启动
- 检查项目是否正确部署
- 检查URL路径是否正确

### 4. 中文乱码
- 确保数据库字符集为 `utf8mb4`
- 确保项目编码为 `UTF-8`
- 检查 `web.xml` 中的字符编码过滤器配置
- **Docker 环境**：确认 `jdbc.url` 使用 `characterEncoding=UTF-8` 与 `connectionCollation=utf8mb4_unicode_ci`（见 `docker/campus-db.properties`）；必要时 `docker compose down -v` 后重新 `up --build`（见第二节「常见问题」）

### 5. Docker 相关
- 构建失败 `invalid flag: -proc:full`：已在本项目 `pom.xml` 中移除对 JDK8 不兼容的编译参数；请拉取最新代码后重试构建。
- **`docker compose` 与 `docker-compose`**：新版推荐使用 **`docker compose`**（带空格）。
